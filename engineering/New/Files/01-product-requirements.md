# 01 — Product Requirements

> **Platform:** AI Smart University Platform
> **Version:** v2.0.0 | **Date:** March 2026
> **Classification:** Confidential — Internal Use Only

---

## 1. Product Vision

> To become the definitive AI-powered university operating system for every Indian institution — eliminating administrative inefficiency, empowering every stakeholder with intelligent insights, and building a fully connected smart campus ecosystem aligned with NEP 2020.

---

## 2. Mission Statement

Deliver an integrated, multi-tenant, AI-driven academic and administrative platform that unifies student lifecycle management, faculty operations, departmental governance, university administration, and a unique per-institution public portal — all under a single, secure, scalable SaaS infrastructure.

---

## 3. Problem Statement

| Problem | Impact |
|---|---|
| Manual scheduling takes 2–4 weeks per semester | Wasted admin time; error-prone; cascading conflicts |
| Disconnected systems (attendance, fees, library, HR, exams) | No single source of truth; data duplication |
| No student intelligence | Cannot predict dropout risk or personalise learning paths |
| Public transparency gap | Result verification, admissions, scholarship status are opaque |
| NEP 2020 non-compliance | Multidisciplinary, FYUP, credit-flexibility requirements ignored |
| Zero smart campus readiness | IoT, face recognition, digital identity not integrated |
| Manual paperwork | Bonafide certificates, transcripts, NAAC reports generated by hand |

## 7. Non-Functional Requirements

### 7.1 Performance

| NFR ID | Requirement | Target |
|---|---|---|
| NFR-P01 | Timetable generation time | < 30 seconds (≤ 20 faculty, ≤ 10 courses, ≤ 2 batches) |
| NFR-P02 | Page load time (initial) | < 3 seconds on 4G |
| NFR-P03 | API response time (p95) | < 500ms read, < 1s write |
| NFR-P04 | PDF generation time | < 5 seconds |
| NFR-P05 | Real-time update propagation | < 2 seconds |
| NFR-P06 | Concurrent users (MVP) | 500 simultaneous users |
| NFR-P07 | Concurrent users (Scale) | 100,000 simultaneous users |
| NFR-P08 | Blockchain hash verification | < 3 seconds |
| NFR-P09 | AI chatbot response time | < 5 seconds end-to-end |
| NFR-P10 | Public portal load (unauthenticated) | < 2 seconds |

### 7.2 Security

| NFR ID | Requirement | Standard |
|---|---|---|
| NFR-S01 | Passwords hashed with bcrypt (cost ≥ 12) | OWASP Authentication |
| NFR-S02 | All communications via TLS 1.3 | NIST SP 800-52 |
| NFR-S03 | JWT tokens expire after 8 hours; refresh after 30 days | RFC 6749 |
| NFR-S04 | All user actions logged with timestamp and IP | Audit Compliance |
| NFR-S05 | Cross-tenant data access impossible at API level | Multi-tenant Security |
| NFR-S06 | Firebase Authentication for students/faculty with optional MFA | Firebase Security |
| NFR-S07 | Role-Based Access Control (RBAC) with fine-grained permissions | RBAC Standard |
| NFR-S08 | End-to-end encryption for all messaging | E2EE |
| NFR-S09 | Blockchain academic records — immutable on-chain | Ethereum / Polygon L2 |
| NFR-S10 | Zero Trust architecture across all microservices | Zero Trust NIST |
| NFR-S11 | DPDP Act 2023 compliance for all personal data | Indian Data Protection Law |
| NFR-S12 | AI fraud detection on fee payments and verification requests | In-house ML |
| NFR-S13 | VAPT testing before every major release | ISO 27001 |

## 8. NEP 2020 Compliance

| NEP 2020 Mandate | System Support | Implementation |
|---|---|---|
| Multidisciplinary learning | Cross-department elective scheduling | Courses span multiple departments; interdisciplinary batch support |
| FYUP (4-year UG program) | Multi-year program tracking | Program type selector with FYUP; exit point management per year |
| Credit-based system | Credit-hour enforcement | Weekly hours derived from credits; soft constraint for alignment |
| Flexible curriculum | Dynamic elective slots | Elective pools in batch config; AI assigns based on preference |
| Academic calendar alignment | Semester-based scheduling | Start/end dates, working days, holiday exclusion |
| B.Ed./M.Ed./ITEP support | Program-specific constraints | Configurable constraints per program type |
| FYUP exit points | Year-wise scheduling support | Separate configs for Year 1–4 with shared resource pool |
| Multiple entry/exit options | Student lifecycle tracking | Status management for dropouts, re-entries, lateral entries |
| Holistic assessment | Behaviour ratings + AI analytics | Multi-dimensional performance dashboard |
