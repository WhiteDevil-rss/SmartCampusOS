# 🎓 University ERP — Full-Stack Development Prompt
## "SmartCampus OS" — AI-Powered University Operating System

---

## 🧭 Project Overview

You are building **SmartCampus OS** — a comprehensive, AI-powered university management system built as a **multi-tenant SaaS platform**. This is a complete digital campus ecosystem aligned with NEP 2020, featuring five distinct role-based panels, deep AI integration, smart campus IoT support, blockchain verification, and advanced analytics.

The platform must be **production-grade**, **fully responsive**, and **modular** — each panel and feature must be independently functional yet tightly integrated through a unified data layer.

---

## 🏗️ Tech Stack (Recommended)

### Frontend
- **Framework**: Next.js 14+ (App Router) with TypeScript
- **UI Library**: shadcn/ui + Tailwind CSS
- **State Management**: Zustand + React Query (TanStack Query)
- **Real-time**: Socket.IO or Supabase Realtime
- **Charts/Analytics**: Recharts + D3.js
- **Maps/Tracking**: Leaflet.js or Google Maps API
- **QR Generation**: qrcode.react
- **PDF Generation**: react-pdf / jsPDF
- **Video Player**: Video.js or Plyr

### Backend
- **Runtime**: Node.js with Express.js or Fastify (or Next.js API routes for monorepo)
- **Database**: PostgreSQL (primary) + Redis (caching/sessions) + MongoDB (unstructured logs)
- **ORM**: Prisma
- **Auth**: Firebase Auth + JWT with refresh token rotation
- **File Storage**: Firebase Storage or AWS S3
- **AI Integration**: OpenAI API (GPT-4o) + Langchain.js
- **Blockchain**: Ethereum (Polygon) or Hyperledger Fabric for academic records
- **Search**: Meilisearch or Algolia
- **Queue/Jobs**: BullMQ with Redis
- **Email/SMS**: Nodemailer + Twilio / MSG91

### DevOps
- **Containerisation**: Docker + Docker Compose
- **CI/CD**: GitHub Actions
- **Hosting**: Vercel (frontend) + Railway/Render/AWS (backend)
- **Monitoring**: Sentry + PostHog

## 🔐 Authentication & Security System

Implement a **Zero-Trust, Role-Based Access Control (RBAC)** system with the following roles:

```
SUPER_ADMIN > UNIVERSITY_ADMIN > COLLEGE_ADMIN > DEPARTMENT_HEAD >
FACULTY > STUDENT > PARENT > LIBRARIAN > PLACEMENT_OFFICER > PUBLIC
```

### Requirements:
- **Firebase Authentication** as the primary identity provider
- **Multi-Factor Authentication (MFA)** — TOTP + SMS OTP
- **JWT Access Tokens** (15 min) + **Refresh Tokens** (7 days), stored in HttpOnly cookies
- **Role-based route guards** on both frontend (middleware) and backend (middleware)
- **End-to-End Encryption** for messages and sensitive documents
- **Blockchain storage** for academic records (immutable audit trail)
- **Audit Trail System** — log every create/update/delete with user ID, timestamp, IP, and action
- **AI Fraud Detection** — flag anomalous login patterns, mass-grade changes, or suspicious activity
- **Secure API Gateway** — rate limiting, CORS, helmet.js, input sanitisation
- **Data Redundancy + Disaster Recovery** — automated daily DB snapshots, multi-region backup
- **Regulatory Compliance** — FERPA / DPDP Act (India) compliant data handling

## 🤖 AI SMART FEATURES

### AI Timetable Generation Engine
- **Algorithm**: Constraint Programming Solver (CSP using backtracking + arc consistency)
- **Inputs**: Faculty availability, subject credits, room capacity, lab requirements, student electives, holiday calendar
- **Constraints Hard**: No faculty clash, no room double-booking, lab subjects need lab rooms
- **Constraints Soft**: Faculty preference for morning/evening, subject distribution across week
- **AI Workload Balancing**: Distribute teaching load fairly using scoring function
- **AI Fairness Scoring**: Ensure no faculty is disproportionately burdened
- **Elective Conflict Resolution**: AI resolves elective slot clashes for student groups
- **Output**: Published timetable pushed to Student, Faculty, and Department panels

### Predictive Analytics Engine (Python microservice or OpenAI function calls)
- **Student Performance Prediction**: Predict end-semester marks based on internals and attendance
- **Dropout Risk Prediction**: Flag at-risk students using ML model (Random Forest / XGBoost)
- **Course Demand Prediction**: Forecast elective enrollment using historical trends
- **Predictive Enrollment Forecasting**: University-level intake predictions

### AI Academic Advisor
- Conversational advisor for academic planning
- Recommends electives based on career goals and performance
- Flags credit shortfalls and suggests remediation
- Integrates with career recommendation engine

### AI Campus Analytics
- Anomaly detection in campus data (attendance spikes, fee collection drops)
- Natural language report generation: "Generate a summary of this semester's academic performance for Department X"
- Dashboard Q&A: "Which subjects have the most failures this semester?"

### AI Scheduling Optimisation
- Optimise exam timetable to minimise student exam clashes
- Optimise bus route scheduling based on student location data
- Optimise faculty meeting scheduling

- **Primary Palette**: Deep Indigo (#1e1b4b) + Electric Blue (#3b82f6) + Clean White (#ffffff)
- **Accent**: Emerald Green (#10b981) for success, Amber (#f59e0b) for warnings, Red (#ef4444) for danger
- **Typography**: `Inter` (body) + `Cal Sans` or `Instrument Serif` (headings)
- **Border Radius**: 8px (cards), 4px (inputs), 9999px (badges/pills)
- **Shadows**: Subtle, layered depth (no harsh drop shadows)
- **Icons**: Lucide React (consistent 24px stroke icons)

### Layout System
- **Sidebar Navigation**: Collapsible sidebar with role-based menu items
- **Breadcrumb**: Always show current location in hierarchy
- **Top Bar**: Search (global), notifications, profile menu, role switcher (for admins)
- **Content Area**: Max-width container, responsive grid
- **Mobile**: Bottom tab bar (5 key items) + hamburger for full menu

### Reusable Component Library (build these first):
- `<StatCard>` — KPI card with icon, value, trend indicator
- `<DataTable>` — sortable, filterable, paginated table with row actions
- `<FileUpload>` — drag-and-drop with progress bar and preview
- `<StatusBadge>` — colour-coded status indicators
- `<Timeline>` — vertical event timeline
- `<AttendanceGrid>` — calendar-style attendance heatmap
- `<GradeCard>` — subject grade display with breakdown
- `<NotificationPanel>` — grouped, categorised notification list
- `<ChatBubble>` — message UI component
- `<PDFViewer>` — inline PDF display
- `<VideoPlayer>` — custom styled video player
- `<QRDisplay>` — QR code with download option
- `<AIChat>` — chatbot UI with typing indicators

## 🔔 NOTIFICATION SYSTEM

Implement a multi-channel notification system:
- **In-App**: Real-time bell icon with grouped notifications
- **Push Notifications**: Firebase Cloud Messaging (FCM) for mobile
- **Email**: Nodemailer with HTML templates (transactional)
- **SMS**: Twilio / MSG91 for critical alerts (fees due, exam dates)
- **WhatsApp**: (Optional) WhatsApp Business API for fee reminders

Notification categories:
- Academic (grades published, assignment due, timetable change)
- Attendance (low attendance alert, flag approval)
- Fees (due reminders, payment confirmation)
- Library (due date, fine added)
- Events (upcoming events, registration open)
- Placement (new drive posted, application status)
- System (service request status, complaint update)

All responses follow:
```json
{
  "success": true,
  "data": { ... },
  "message": "Operation successful",
  "meta": { "page": 1, "total": 100, "limit": 20 }
}
```

```json
{
  "success": false,
  "error": "RESOURCE_NOT_FOUND",
  "message": "Student with ID 12345 not found",
  "statusCode": 404
}
```
